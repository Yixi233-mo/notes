## Python 模块（Module）学习笔记
### 一、模块概述
**模块（Module）** 是一个以 `.py` 结尾的 Python 源文件，其中可以包含变量、函数、类等。

| 优点 | 说明 |
| --- | --- |
| **提高可维护性** | 将特定功能的代码集中在一个文件中 |
| **提高代码复用性** | 编写一次，多处导入使用 |
| **避免命名冲突** | 相同名字的变量/函数可以存在于不同模块中 |
| **便于组织代码** | 按功能划分，结构清晰 |


> 💡 **模块的本质**：模块就是包含 Python 代码的文件，文件名即模块名。
>

### 二、创建模块
**命名规则**：

+ 模块名区分大小写
+ 不能与 Python 自带的标准模块重名
+ 遵循标识符命名规范（通常使用小写字母和下划线）

**创建 **`my_add.py`** 模块**：

```python
# my_add.py
"""
这是一个简单的加法模块
"""

num = 100

def add(a, b):
    """求两个数的和"""
    return a + b
```

### 三、导入模块
#### 3.1 全部导入 `import`
导入模块的所有成员，通过 `模块名.成员名` 的方式访问。

**语法**：

```python
import 模块名 [as 别名]
```

**案例**：

```python
# main.py
# 导入模块
import my_add

# 使用模块中的成员
print(my_add.add(1, 2))   # 3
print(my_add.num)         # 100

# 使用别名
import my_add as a1
print(a1.add(1, 2))       # 3
print(a1.num)             # 100
```

**特点**：

+ 即使多次 `import`，模块也只被导入一次
+ 需要通过模块名访问成员，避免命名冲突

#### 3.2 局部导入 `from ... import`
指定导入模块的部分成员，直接通过**成员名**访问。

**语法**：

```python
from 模块名 import 成员名1 [as 别名], 成员名2 [as 别名], ...
```

**创建 **`my_multi.py`** 模块**：

```python
# my_multi.py
num = 200
_str1 = "abc"

def multi(a, b):
    """求两个数的积"""
    return a * b
```

**案例**：

```python
# 只能使用导入的成员
from my_add import add
print(add(1, 2))      # 3
# print(num)          # ❌ NameError: name 'num' is not defined

# 使用别名区分同名变量
from my_add import add, num as a1
from my_multi import num as m1

print(add(1, 2))      # 3
print(a1)             # 100（来自 my_add）
print(m1)             # 200（来自 my_multi）
```

**⚠️**** 注意**：如果多个模块中存在重名成员，后一次导入会覆盖前一次导入。

```python
from my_add import num      # 导入 num = 100
from my_multi import num    # 覆盖：num = 200

print(num)  # 200（来自 my_multi）
```

#### 3.3 局部导入 `from ... import *`
导入模块中所有**不以单下划线开头**的成员，直接通过成员名访问。

**语法**：

```python
from 模块名 import *
```

**案例**：

```python
# 导入模块所有非私有成员
from my_add import *

print(add(1, 2))   # 3
print(num)         # 100
# print(_str1)     # ❌ 以下划线开头的成员不会被导入
```

> ⚠️ **不推荐**：`from ... import *` 可能导致命名冲突，且代码可读性差。建议只在交互式环境中使用。
>

### 四、模块搜索顺序
当导入一个模块时，Python 会按以下顺序搜索：

| 顺序 | 搜索位置 |
| --- | --- |
| 1 | **当前目录** |
| 2 | **PYTHONPATH** 环境变量中的目录 |
| 3 | **标准库目录**（Python 安装目录） |


#### 查看搜索路径
```python
import sys

# 查看所有模块搜索路径
print(sys.path)

# 示例输出：
# ['当前目录', 'PYTHONPATH目录', '标准库目录', ...]
```

#### 临时添加搜索路径
```python
import sys

# 添加当前目录的父目录到搜索路径
sys.path.append("./..")

# 再次查看
print(sys.path)
```

> 💡 若需要永久添加，可以设置 `PYTHONPATH` 环境变量。
>

### 五、`__all__` 控制导入内容
`__all__` 用于控制 `from ... import *` 时哪些成员可以被导入。

**在 **`my_add.py`** 中设置**：

```python
# my_add.py
__all__ = ["num", "add"]   # 只允许导入 num 和 add

num = 100
num1 = 200                 # 不在 __all__ 中
_str1 = "abc"              # 以下划线开头，默认不导入

def add(a, b):
    return a + b
```

**使用 **`from ... import *`：

```python
from my_add import *

print(add(1, 2))   # 3（可以访问）
print(num)         # 100（可以访问）
# print(num1)      # ❌ NameError: name 'num1' is not defined
```

**使用 **`import`** 不受 **`__all__`** 限制**：

```python
import my_add

print(my_add.add(1, 2))   # 3
print(my_add.num)         # 100
print(my_add.num1)        # 200（可访问）
```

> 💡 `__all__`** 的作用**：
>
> + 限制 `from ... import *` 的行为
> + 明确模块的公共接口
> + 对 `import 模块名` 方式无效
>

### 六、`__name__` 内置变量
`__name__` 是 Python 的特殊内置变量，用于判断模块的运行方式：

| 运行方式 | `__name__` 的值 |
| --- | --- |
| 直接运行 | `"__main__"` |
| 作为模块导入 | 模块名（如 `"my_add"`） |


#### 6.1 问题：模块中的测试代码被执行
**在 **`my_add.py`** 中**：

```python
# my_add.py
def add(a, b):
    return a + b

# 测试代码
print(add(10, 20))   # 直接运行时正常执行
```

**在 **`main.py`** 中导入**：

```python
# main.py
import my_add

# 输出：30（测试代码被执行了！）
```

#### 6.2 解决方案：使用 `if __name__ == "__main__"`
```python
# my_add.py
def add(a, b):
    return a + b

print(__name__)   # 直接运行时为 "__main__"，导入时为 "my_add"

if __name__ == "__main__":
    # 只有在直接运行此文件时才会执行
    print(add(10, 20))
```

**效果**：

+ 直接运行 `my_add.py`：输出 `__main__` 和 `30`
+ 在 `main.py` 中导入：不输出任何内容

> 💡 **最佳实践**：在模块中，将测试代码放在 `if __name__ == "__main__":` 下。
>

### 七、模块的综合示例
**创建 **`calculator.py`** 模块**：

```python
# calculator.py
"""
计算器模块
提供基本的数学运算功能
"""

__all__ = ["add", "subtract", "multiply", "divide"]

# 常量
PI = 3.14159
VERSION = "1.0.0"
_DEBUG = False          # 私有常量，不会被 import * 导入

def add(a, b):
    """加法"""
    return a + b

def subtract(a, b):
    """减法"""
    return a - b

def multiply(a, b):
    """乘法"""
    return a * b

def divide(a, b):
    """除法"""
    if b == 0:
        raise ValueError("除数不能为0")
    return a / b

def _internal_validate(x):
    """私有方法：验证输入（不会被 import * 导入）"""
    return isinstance(x, (int, float))

# 测试代码
if __name__ == "__main__":
    print("计算器模块测试：")
    print(f"3 + 5 = {add(3, 5)}")
    print(f"10 - 4 = {subtract(10, 4)}")
    print(f"6 * 7 = {multiply(6, 7)}")
    print(f"15 / 3 = {divide(15, 3)}")
    print(f"PI = {PI}")
```

**在 **`main.py`** 中使用**：

```python
# main.py
import calculator as calc

# 使用模块中的常量和方法
print(f"版本: {calc.VERSION}")
print(f"PI: {calc.PI}")
print(f"3 + 5 = {calc.add(3, 5)}")
print(f"15 / 3 = {calc.divide(15, 3)}")

# 查看模块的文档
print(calc.__doc__)

# 查看模块中的公共成员
print(dir(calc))
# ['PI', 'VERSION', '__all__', '__builtins__', '__cached__', ...]
```

### 八、易错点总结
| 易错点 | 正确理解 |
| --- | --- |
| 模块名与变量/函数同名 | ✅ 导入时使用别名可避免冲突 |
| `from ... import *` 导入所有成员 | ⚠️ 不导入以下划线开头的成员 |
| 模块导入后重新加载 | ❌ 模块只会被导入一次，需用 `importlib.reload()` |
| `__all__` 限制所有导入方式 | ❌ 仅对 `from ... import *` 有效 |
| 忘记 `__name__ == "__main__"` | ❌ 测试代码会在模块被导入时执行 |
| 模块搜索路径问题 | 使用 `sys.path.append()` 临时添加 |


### 九、快速参考卡片
```python
# === 导入方式 ===

# 1. 全部导入
import 模块名
import 模块名 as 别名
模块名.成员名

# 2. 局部导入
from 模块名 import 成员名
from 模块名 import 成员名 as 别名
from 模块名 import 成员名1, 成员名2

# 3. 全部导入（不推荐）
from 模块名 import *

# === 模块控制 ===

# 控制 from ... import * 的行为
__all__ = ["成员1", "成员2"]   # 只导入指定的公共成员

# 控制测试代码
if __name__ == "__main__":
    # 只在直接运行时执行

# === 搜索路径 ===
import sys
print(sys.path)                      # 查看搜索路径
sys.path.append("/自定义路径")        # 临时添加

# === 模块信息 ===
模块.__doc__          # 模块文档字符串
模块.__file__         # 模块文件路径
dir(模块)             # 查看模块的成员列表
help(模块)            # 查看模块帮助信息
```

> 📌 **核心记忆**：
>
> + **模块**是以 `.py` 结尾的 Python 文件，用于组织代码
> + `import 模块名`：全部导入，通过 `模块名.成员` 访问
> + `from 模块名 import 成员`：局部导入，直接通过成员名访问
> + `__all__`：控制 `from ... import *` 的行为
> + `__name__`：直接运行时为 `"__main__"`，导入时为模块名
> + **最佳实践**：模块中的测试代码放在 `if __name__ == "__main__":` 下
>
