一、线程的基本概念 ​

线程是处理器任务调度和执行的基本单位，是进程内的一个执行单元。

特性说明
隶属关系一个进程至少有一个线程，可运行多个线程
资源共享多个线程之间可共享数据（共享进程的内存空间）
稳定性线程异常未捕获会导致整个进程崩溃
调度单位线程是 CPU 调度的基本单位

进程 vs 线程：

plain
┌─────────────────────────────────────────────────────────┐
│                        进程                             │
│  ┌──────────┬──────────┬──────────┬──────────────┐      │
│  │  代码段   │  数据段   │  堆      │  文件描述符  │      │
│  └──────────┴──────────┴──────────┴──────────────┘      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│  │ 线程1    │  │ 线程2    │  │ 线程3    │                │
│  │ (栈)     │  │ (栈)     │  │ (栈)     │                │
│  └──────────┘  └──────────┘  └──────────┘          │
└─────────────────────────────────────────────────────────┘

二、使用 threading.Thread 创建线程 ​

Python 提供了 _thread（低级）和 threading（高级）两个模块。通常使用 threading 模块。

2.1 Thread 类参数 ​

python
threading.Thread(group=None, target=None, name=None, args=(), kwargs={}, *, daemon=None)
1
参数说明
group应为 None（保留扩展）
target线程执行的可调用对象
name线程名称（默认 "Thread-N"）
args目标函数的参数元组
kwargs目标函数的关键字参数字典
daemon是否为守护线程

2.2 Thread 常用属性和方法 ​

属性/方法说明
name线程名称
daemon是否为守护线程
ident线程标识符
native_id操作系统分配的线程 ID
start()启动线程（调用 run()）
run()线程执行的方法（可被子类重写）
join([timeout])阻塞等待线程结束
is_alive()检查线程是否在运行
threading.enumerate()获取当前所有存活线程列表
threading.current_thread()获取当前线程实例

2.3 基本示例 ​

python
import time
import threading

def func():
    """线程执行的任务"""
    flag = 0
    while True:
        current = threading.current_thread()
        print(f"{current.name}: {str(flag) * 5}")
        flag = flag ^ 1   # 0 和 1 交替
        time.sleep(0.5)

if __name__ == "__main__":
    t1 = threading.Thread(target=func, name="线程1")
    t2 = threading.Thread(target=func, name="线程2")
    
    t1.start()
    t2.start()

三、自定义 Thread 子类 ​

通过继承 Thread 类并重写 run() 方法创建线程。

python
import time
import threading

class Worker(threading.Thread):
    def __init__(self, name):
        super().__init__()
        self.name = name
    
    def run(self):
        """线程执行的任务"""
        flag = 0
        while True:
            print(f"\r{self.name}: {str(flag) * 5}", end="")
            flag = flag ^ 1
            time.sleep(0.2)

if __name__ == "__main__":
    t1 = Worker("线程1")
    t2 = Worker("线程2")
    t1.start()
    t2.start()

四、线程池（ThreadPoolExecutor） ​

ThreadPoolExecutor 是 concurrent.futures 模块中的线程池实现。

4.1 创建线程池 ​

python
concurrent.futures.ThreadPoolExecutor(max_workers=None, thread_name_prefix="")
1
参数说明
max_workers最大线程数（默认由系统决定）
thread_name_prefix线程名称前缀

4.2 常用方法 ​

方法说明
submit(fn, *args, **kwargs)提交任务，返回 Future 对象
map(func, *iterables)并行执行 map，返回结果迭代器
shutdown(wait=True)关闭线程池（with 语句自动调用）

4.3 示例 ​

python
import concurrent.futures

def encrypt(text, thread_name):
    """加密：每个字符与 1 异或"""
    chars = list(text)
    for i, char in enumerate(chars):
        chars[i] = chr(ord(char) ^ 1)
        print(f"{thread_name}: {''.join(chars)}")
    return ''.join(chars)

if __name__ == "__main__":
    text = list("idmmn!vnsme")  # 加密前的文本
    
    # 使用 with 语句自动管理线程池
    with concurrent.futures.ThreadPoolExecutor(max_workers=3) as executor:
        future1 = executor.submit(encrypt, text, "线程1")
        future2 = executor.submit(encrypt, text, "线程2")
        future3 = executor.submit(encrypt, text, "线程3")
        
        # 获取结果
        result1 = future1.result()
        result2 = future2.result()
        result3 = future3.result()
    
    print("最终结果:", result1)  # hello world

5.1 线程安全问题 ​

多个线程同时修改共享数据时，可能产生数据不一致的问题。

python
import time
import threading

g_num = 0

def func():
    global g_num
    for _ in range(10):
        tmp = g_num + 1
        time.sleep(0.01)  # 模拟耗时操作
        g_num = tmp

if __name__ == "__main__":
    threads = [threading.Thread(target=func) for _ in range(3)]
    for t in threads:
        t.start()
    for t in threads:
        t.join()
    
    print(g_num)  # 预期 30，实际可能是 10 或其他值

5.2 使用互斥锁 ​

Lock 保证每次只有一个线程能修改共享数据。

python
import time
import threading

g_num = 0
lock = threading.Lock()  # 创建锁

def func():
    global g_num
    for _ in range(10):
        lock.acquire()    # 获取锁
        tmp = g_num + 1
        time.sleep(0.01)
        g_num = tmp
        lock.release()    # 释放锁

if __name__ == "__main__":
    threads = [threading.Thread(target=func) for _ in range(3)]
    for t in threads:
        t.start()
    for t in threads:
        t.join()
    
    print(g_num)  # 30（正确）

六、GIL（全局解释器锁） ​

6.1 什么是 GIL ​

GIL（Global Interpreter Lock） 是 CPython 解释器中的一个互斥锁，同一时刻只允许一个线程执行 Python 字节码。

plain
GIL 的工作机制：

线程1 ────┐    ┌────────┐    ┌────────┐
线程2 ────┼────┼────────┼────┼────────┼───→ 时间
线程3 ────┘    └────────┘    └────────┘

同一时刻只有一个线程在执行 Python 代码

6.2 GIL 的历史原因 ​

  • Python 诞生于 1991 年，当时操作系统还没有线程概念
  • 多核时代到来后，Python 开始支持多线程
  • GIL 提供了线程安全的内存管理（引用计数）

6.3 GIL 对性能的影响 ​

任务类型GIL 影响原因
CPU 密集型⚠️ 严重多个线程无法利用多核
I/O 密集型✅ 较小I/O 操作时释放 GIL
混合型⚠️ 部分影响取决于 CPU 计算占比

6.4 GIL 的调度机制 ​

plain
原始机制：
1. 计算当前执行的字节码数量
2. 达到阈值后强制释放 GIL
3. 触发操作系统线程调度
4. 主线程可能很快再次获取 GIL
5. 其他线程在等待中浪费 CPU 时间

改进方向：
- 在 I/O 等待时主动释放 GIL
- 优化 GIL 的获取和释放策略
- Python 各版本持续改进 GIL 机制

6.5 绕过 GIL 的方法 ​

方法说明
多进程使用 multiprocessing，每个进程有独立 GIL
C 扩展NumPy 等库在 C 层释放 GIL
异步 I/O使用 asyncio 处理 I/O 密集型任务
其他 Python 实现PyPy、Jython 等有不同的 GIL 策略
python
# CPU 密集型任务：多线程受 GIL 影响
import threading
import time

def cpu_heavy():
    total = 0
    for i in range(10000000):
        total += i ** 2
    return total

# 多线程（受 GIL 限制）
start = time.time()
threads = [threading.Thread(target=cpu_heavy) for _ in range(4)]
for t in threads: t.start()
for t in threads: t.join()
print(f"多线程耗时: {time.time() - start:.2f}s")

# 多进程（绕过 GIL）
import multiprocessing
start = time.time()
with multiprocessing.Pool(4) as pool:
    pool.map(lambda _: cpu_heavy(), range(4))
print(f"多进程耗时: {time.time() - start:.2f}s")

七、线程 vs 进程使用场景 ​

场景推荐方案原因
CPU 密集型计算多进程不受 GIL 限制，利用多核
I/O 密集型操作多线程切换开销小，I/O 时释放 GIL
网络请求/爬虫多线程/异步等待响应时切换执行
文件读写多线程I/O 操作释放 GIL
科学计算多进程 + NumPyNumPy 可释放 GIL

八、综合示例：多线程下载器 ​

python
"""
多线程文件下载器
演示线程池和互斥锁的使用
"""

import threading
import time
import random
from concurrent.futures import ThreadPoolExecutor, as_completed

class Downloader:
    def __init__(self, max_workers=3):
        self.max_workers = max_workers
        self.completed = 0
        self.total = 0
        self.lock = threading.Lock()
        self.results = []
    
    def download_file(self, file_id, size):
        """模拟下载文件"""
        thread_name = threading.current_thread().name
        print(f"[{thread_name}] 开始下载文件 {file_id} ({size}MB)")
        
        # 模拟下载过程
        downloaded = 0
        while downloaded < size:
            chunk = random.randint(1, 10)
            downloaded += chunk
            if downloaded > size:
                downloaded = size
            progress = (downloaded / size) * 100
            print(f"[{thread_name}] 文件 {file_id} 下载进度: {progress:.1f}%")
            time.sleep(random.random() * 0.5)
        
        print(f"[{thread_name}] 文件 {file_id} 下载完成")
        
        # 更新完成计数（使用锁保护）
        with self.lock:
            self.completed += 1
            self.results.append(f"文件{file_id}")
        
        return f"文件 {file_id} 下载成功"
    
    def run(self, file_list):
        """执行下载任务"""
        self.total = len(file_list)
        self.completed = 0
        
        print(f"=== 开始下载 {self.total} 个文件 ===")
        
        with ThreadPoolExecutor(max_workers=self.max_workers) as executor:
            # 提交所有任务
            futures = {
                executor.submit(self.download_file, file_id, size): file_id
                for file_id, size in file_list
            }
            
            # 获取结果
            for future in as_completed(futures):
                file_id = futures[future]
                try:
                    result = future.result()
                    print(f"✅ {result}")
                except Exception as e:
                    print(f"❌ 文件 {file_id} 下载失败: {e}")
        
        print(f"\n=== 所有文件下载完成，共 {len(self.results)} 个文件 ===")
        return self.results


if __name__ == "__main__":
    # 模拟文件列表：(文件ID, 文件大小MB)
    files = [
        (1, 15),
        (2, 8),
        (3, 20),
        (4, 12),
        (5, 6),
        (6, 25),
    ]
    
    downloader = Downloader(max_workers=3)
    results = downloader.run(files)
    
    print("\n下载完成的文件:", results)

九、易错点总结 ​

易错点正确理解
多线程一定能提升性能CPU 密集型任务受 GIL 限制，多线程可能更慢
线程间数据共享无需加锁❌ 需要互斥锁保护共享数据
忘记 join() 等待线程结束主线程可能提前退出，子线程被强制终止
守护线程和普通线程一样守护线程在主线程结束时自动终止
GIL 是 Python 的缺陷GIL 是 CPython 的实现细节，保证了内存安全
I/O 密集型任务也受 GIL 限制I/O 操作时会释放 GIL，影响较小

📌 核心记忆：

  • 线程是调度执行的基本单位，共享进程资源
  • 多线程适合 I/O 密集型任务，多进程适合 CPU 密集型任务
  • GIL 是 CPython 的限制，同一时刻只有一个线程执行 Python 代码
  • 互斥锁（Lock） 保护共享数据，防止线程安全问题
  • join() 等待线程结束，确保数据完整性
  • 线程池用 ThreadPoolExecutor，配合 with 语句使用
