一、面向过程 vs 面向对象 ​

1.1 两种编程范式 ​

范式核心思想关注点适合场景
面向过程编程以过程（函数）为中心解决问题的步骤和流程简单、线性任务
面向对象编程以对象为中心系统中的对象及对象间的交互复杂、大型系统

Python 是混合型语言：既支持面向过程，也支持面向对象。

1.3 面向对象举例：做晚餐 ​

python
class Dish:
    """菜品类"""
    def __init__(self, name):
        self.name = name
    
    def prepare(self):
        pass

class Salad(Dish):
    """沙拉类"""
    def prepare(self):
        print(f"为 {self.name} 购买食材。")
        print(f"清洗 {self.name} 的蔬菜。")
        print(f"切 {self.name} 的蔬菜。")

class Stew(Dish):
    """炖菜类"""
    def prepare(self):
        print(f"为 {self.name} 购买食材。")
        print(f"切 {self.name} 的肉。")
        print(f"烹饪 {self.name}。")

# 创建菜品对象并准备
salad = Salad("蔬菜沙拉")
stew = Stew("炖肉")

salad.prepare()
stew.prepare()

二、类与对象的基本概念 ​

2.1 类（Class） ​

定义说明
是什么对大量对象共性的抽象

七、属性详解 ​

7.1 类属性（类变量） ​

定义位置：在类中、方法外定义。

特点：

  • 所有实例共享同一个类属性
  • 可通过 类名.属性名 或 实例名.属性名 访问
  • 通过 类名.属性名 修改会影响所有实例
  • 通过 实例名.属性名 赋值会创建实例属性（不推荐同名）
python
class Person:
    home = "earth"  # 类属性

# 访问
print(Person.home)  # earth

p1 = Person()
p2 = Person()
print(p1.home)      # earth
print(p2.home)      # earth

# 通过类名修改（影响所有实例）
Person.home = "mars"
print(p1.home)      # mars
print(p2.home)      # mars

# 通过实例名修改（创建实例属性，不推荐）
p1.home = "venus"   # 创建实例属性，不影响类
print(Person.home)  # mars（类属性不变）
print(p1.home)      # venus（实例属性）
print(p2.home)      # mars（仍使用类属性）

八、方法详解 ​

Python 类中有三种方法：实例方法、类方法、静态方法。

方法类型定义装饰器第一个参数调用方式可访问内容
实例方法无self（实例）实例调用实例属性、类属性、其他方法
类方法@classmethodcls（类）类或实例调用类属性、其他类方法
静态方法@staticmethod无类或实例调用仅依赖传入参数

8.1 实例方法 ​

python
class Person:
    home = "earth"
    
    def __init__(self, name):
        self.name = name
    
    def instance_method(self):
        """实例方法：第一个参数为 self"""
        print(f"实例方法被调用: {self.name}")
        print(f"访问类属性: {self.home}")  # 或 Person.home

p = Person("张三")
p.instance_method()  # 通过实例调用
# Person.instance_method(p)  # 等价形式

8.4 三种方法对比示例 ​

python
class Example:
    class_var = "类变量"
    
    def __init__(self, name):
        self.name = name
    
    def instance_method(self):
        """实例方法：可以访问实例属性和类属性"""
        print(f"实例方法: name={self.name}, class_var={self.class_var}")
    
    @classmethod
    def class_method(cls):
        """类方法：只能访问类属性"""
        print(f"类方法: class_var={cls.class_var}")
    
    @staticmethod
    def static_method():
        """静态方法：不能访问实例属性或类属性"""
        print("静态方法: 只能使用传入的参数")

# 调用
obj = Example("test")
obj.instance_method()   # 实例方法: name=test, class_var=类变量
Example.class_method()  # 类方法: class_var=类变量
Example.static_method() # 静态方法: 只能使用传入的参数

8.6 特殊方法（魔法方法） ​

方法说明触发时机
__new__(cls, ...)创建实例实例化时第一个被调用
__init__(self, ...)初始化实例__new__ 之后调用
__del__(self)销毁对象对象被垃圾回收时
__str__(self)字符串表示str(obj) 或 print(obj)
__repr__(self)开发态字符串表示repr(obj)，通常可用于重建对象
__getattribute__(self, name)属性访问拦截访问任何属性时
python
class Person:
    def __init__(self, name):
        self.name = name
    
    def __str__(self):
        """人类可读的输出"""
        return f"Person: {self.name}"
    
    def __repr__(self):
        """机器可读的输出，通常可用于重建对象"""
        return f"Person('{self.name}')"

p = Person("张三")
print(p)           # Person: 张三（调用 __str__）
print(repr(p))     # Person('张三')（调用 __repr__）

9.6 __slots__：限制实例属性 ​

__slots__ 可以限制实例能动态添加的属性和实例方法。

python
import types

class Person:
    __slots__ = ("name", "age", "eat")  # 只允许这些实例属性/方法
    
    def __init__(self, name):
        self.name = name

p = Person("张三")

# 允许添加
p.age = 18
print(p.age)  # 18

# 允许添加实例方法（如果在 __slots__ 中）
def eat(self):
    print(f"{self.name}在吃饭")

p.eat = types.MethodType(eat, p)
p.eat()  # 张三在吃饭

# 禁止添加不在 __slots__ 中的属性
# p.weight = 100  # AttributeError: 'Person' object has no attribute 'weight'

十、综合示例：学生管理系统 ​

python
class Student:
    """学生类"""
    
    # 类属性：学校名称（所有学生共享）
    school = "第一中学"
    
    def __init__(self, name, student_id, score=0):
        """初始化实例属性"""
        self.name = name
        self.id = student_id
        self.score = score
    
    # 实例方法
    def study(self, hours):
        print(f"{self.name} 学习了 {hours} 小时")
        self.score += hours * 5
        print(f"成绩提升到: {self.score}")
    
    def display_info(self):
        """显示学生信息"""
        print(f"姓名: {self.name}")
        print(f"学号: {self.id}")
        print(f"成绩: {self.score}")
        print(f"学校: {self.school}")
    
    # 类方法
    @classmethod
    def change_school(cls, new_school):
        """修改学校名称"""
        cls.school = new_school
        print(f"学校已更名为: {cls.school}")
    
    # 静态方法
    @staticmethod
    def is_pass(score):
        """判断是否及格"""
        return score >= 60


# 使用示例
# 创建学生
stu1 = Student("张三", "2024001", 60)
stu2 = Student("李四", "2024002", 55)

十一、易错点总结 ​

易错点正确理解
类名使用小写类名使用大驼峰命名法
实例方法忘记写 self实例方法第一个参数必须是 self
类属性通过实例修改通过实例修改类属性会创建实例属性，不修改类属性
__init__ 返回非 None__init__ 只能返回 None
混淆类方法和静态方法类方法有 @classmethod 和 cls 参数，可访问类属性；静态方法无特殊参数
忽略 __slots__ 对子类的影响__slots__ 仅对当前类生效，对子类无效

十二、快速参考卡片 ​

python
# 定义类
class ClassName:
    class_var = "类属性"           # 类属性
    
    def __init__(self, param):     # 构造方法
        self.instance_var = param  # 实例属性
    
    def instance_method(self):     # 实例方法
        print(self.instance_var)
    
    @classmethod
    def class_method(cls):         # 类方法
        print(cls.class_var)
    
    @staticmethod
    def static_method():           # 静态方法
        print("静态方法")

# 创建对象
obj = ClassName("值")

# 访问成员
obj.instance_method()           # 调用实例方法
ClassName.class_method()        # 调用类方法
ClassName.static_method()       # 调用静态方法
print(ClassName.class_var)      # 访问类属性
print(obj.instance_var)         # 访问实例属性

# 动态操作
obj.new_attr = "新属性"          # 动态添加属性
del obj.new_attr                # 删除属性

# 限制实例属性
class Restricted:
    __slots__ = ("allowed1", "allowed2")  # 只允许这些属性

📌 核心记忆：

  • 类是模板（图纸），对象是实例（房子）
  • 类名用大驼峰，实例方法第一个参数是 self
  • __init__ 是构造方法，用于初始化实例属性
  • 类属性所有实例共享，实例属性每个实例独有
  • 类方法用 @classmethod（参数 cls），静态方法用 @staticmethod（无特殊参数）
  • Python 支持动态添加属性和方法
  • __slots__ 可以限制实例属性，节省内存
