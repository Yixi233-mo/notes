一、进程的基本概念 ​

进程是操作系统进行资源分配的基本单位，是一个正在运行的程序或软件的实例。

特性说明
资源分配每个进程有独立的内存空间
独立性进程间相互隔离，一个进程崩溃通常不影响其他进程
通信方式需要特殊机制（管道、队列、共享内存等）
系统开销创建和切换进程开销较大

二、使用 multiprocessing.Process 创建进程 ​

2.1 Process 类参数 ​

python
multiprocessing.Process(group=None, target=None, name=None, args=(), kwargs={}, *, daemon=None)
1
参数说明
group始终为 None（兼容 threading.Thread）
target由 run() 调用的可调用对象
name进程名称（自动分配，如不指定）
args目标函数的参数元组
kwargs目标函数的关键字参数字典
daemon是否为守护进程

2.2 Process 常用属性和方法 ​

属性/方法说明
name进程名称
pid进程ID
daemon是否为守护进程
exitcode子进程退出状态码
start()启动进程（只能调用一次）
run()进程执行的方法（可被子类重写）
join([timeout])阻塞等待子进程结束
terminate()强制终止子进程
kill()杀死进程（更彻底）
is_alive()检查进程是否仍在运行

2.3 进程相关函数 ​

函数说明
os.getpid()获取当前进程ID
os.getppid()获取父进程ID

2.4 基本示例 ​

python
import time
import multiprocessing
import os

def write_file():
    """向文件写入数据"""
    with open("test.txt", "a") as f:
        for i in range(10):
            f.write(f"hello world {i}\n")
            f.flush()
            time.sleep(0.5)

def read_file():
    """从文件读取数据"""
    with open("test.txt", "r") as f:
        for _ in range(20):
            time.sleep(0.1)
            print(f.read(1), end="", flush=True)

if __name__ == "__main__":
    # 创建子进程
    p1 = multiprocessing.Process(target=write_file)
    p2 = multiprocessing.Process(target=read_file)
    
    # 启动子进程
    p1.start()
    p2.start()
    
    # 等待子进程结束
    p1.join()
    p2.join()
    
    print("\n主进程结束")

三、自定义 Process 子类 ​

通过继承 Process 类并重写 run() 方法创建进程。

python
import os
import multiprocessing

class Worker(multiprocessing.Process):
    def __init__(self, name=None):
        super().__init__(name=name)
    
    def run(self):
        """进程执行的任务"""
        print(f"进程名: {self.name}")
        print(f"进程ID: {os.getpid()}")
        print(f"父进程ID: {os.getppid()}")

if __name__ == "__main__":
    workers = []
    for i in range(5):
        p = Worker(name=f"Worker-{i+1}")
        p.start()
        workers.append(p)
    
    for p in workers:
        p.join()

四、进程池（Pool） ​

当需要启动大量子进程时，使用进程池可以管理进程的生命周期，避免频繁创建和销毁进程的开销。

4.1 进程池创建 ​

python
multiprocessing.Pool(processes=None, initializer=None, initargs=(), maxtasksperchild=None)
1
参数说明
processes工作进程数量（默认为 CPU 核心数）
initializer每个工作进程启动时调用的初始化函数
initargs初始化函数的参数
maxtasksperchild每个进程完成的最大任务数后自动重启

4.2 进程池常用方法 ​

方法说明
apply(func[, args[, kwds]])同步执行，阻塞等待结果
apply_async(func[, args[, kwds[, callback]]])异步执行，立即返回
map(func, iterable[, chunksize])并行执行，返回结果列表
map_async(func, iterable[, chunksize[, callback]])异步 map
close()不再接受新任务
terminate()立即终止所有工作进程
join()等待工作进程结束（需先调用 close()）

4.3 进程池示例 ​

python
import os
import time
import multiprocessing

def task(n):
    """执行任务的函数"""
    pid = os.getpid()
    print(f"进程 {pid} 开始执行任务 {n}")
    time.sleep(0.5)
    print(f"进程 {pid} 完成任务 {n}")
    return n ** 2

if __name__ == "__main__":
    # 创建进程池（5个工作进程）
    pool = multiprocessing.Pool(processes=5)
    
    # 方式1：同步执行
    print("=== 同步执行 apply ===")
    for i in range(5):
        result = pool.apply(task, args=(i,))
        print(f"结果: {result}")
    
    # 方式2：异步执行
    print("\n=== 异步执行 apply_async ===")
    results = []
    for i in range(5, 10):
        result = pool.apply_async(task, args=(i,))
        results.append(result)
    
    # 获取异步结果
    for result in results:
        print(f"异步结果: {result.get()}")
    
    # 方式3：使用 map（推荐）
    print("\n=== 使用 map ===")
    results = pool.map(task, range(10, 15))
    print(f"map 结果: {results}")
    
    # 关闭进程池
    pool.close()
    pool.join()
    print("所有任务完成")

五、进程间通信（IPC） ​

进程间不共享全局变量，需要使用专门的通信机制。

5.1 进程间不共享全局变量 ​

python
import os
import multiprocessing

def add_elements(lst):
    for i in range(5):
        lst.append(i)
        print(f"子进程 {os.getpid()}: {lst}")

if __name__ == "__main__":
    shared_list = []  # 主进程的列表
    
    p1 = multiprocessing.Process(target=add_elements, args=(shared_list,))
    p2 = multiprocessing.Process(target=add_elements, args=(shared_list,))
    
    p1.start()
    p2.start()
    p1.join()
    p2.join()
    
    print(f"主进程 {os.getpid()}: {shared_list}")  # 输出: []（未被修改）

5.2 使用 Queue 通信 ​

multiprocessing.Queue 用于进程间安全地传递数据。

常用方法：

方法说明
put(obj[, block[, timeout]])放入对象（可设置阻塞）
put_nowait(obj)非阻塞放入
get([block[, timeout]])取出对象（可设置阻塞）
get_nowait()非阻塞取出
qsize()队列大致长度（不可靠）
empty()是否为空（不可靠）
full()是否已满（不可靠）

示例：

python
import time
import random
import multiprocessing

def producer(queue):
    """生产者：向队列中放入随机数"""
    for i in range(10):
        num = random.randint(1, 50)
        queue.put(num)
        print(f"生产者: 放入 {num}")
        time.sleep(random.random())

def consumer(queue):
    """消费者：从队列中取出数据"""
    while True:
        try:
            num = queue.get(timeout=1)
            print(f"消费者: 取出 {num}, 打印 {'=' * num}")
        except:
            break

if __name__ == "__main__":
    queue = multiprocessing.Queue()
    
    p1 = multiprocessing.Process(target=producer, args=(queue,))
    p2 = multiprocessing.Process(target=consumer, args=(queue,))
    
    p1.start()
    p2.start()
    
    p1.join()
    p2.join()

5.3 进程池中使用 Manager().Queue ​

multiprocessing.Queue 与进程池存在兼容性问题，应使用 Manager().Queue。

python
import time
import random
import multiprocessing

def producer(queue):
    for i in range(10):
        queue.put(random.randint(1, 50))
        time.sleep(random.random())

def consumer(queue):
    while True:
        try:
            num = queue.get(timeout=1)
            print("=" * num)
        except:
            break

if __name__ == "__main__":
    # 使用 Manager 创建跨进程队列
    manager = multiprocessing.Manager()
    queue = manager.Queue()
    
    pool = multiprocessing.Pool(2)
    pool.apply_async(producer, (queue,))
    pool.apply_async(consumer, (queue,))
    
    pool.close()
    pool.join()

六、进程与线程综合对比 ​

维度多进程（multiprocessing）多线程（threading）
内存空间独立内存空间共享内存空间
数据共享需要 IPC（Queue、Pipe 等）共享全局变量（需加锁）
创建开销较大较小
切换开销较大较小
GIL 影响不受 GIL 限制受 GIL 限制
CPU 密集型✅ 适合❌ 不适合
I/O 密集型可以，但开销大✅ 适合
隔离性进程间相互独立线程间相互影响

七、易错点总结 ​

易错点正确理解
进程间共享全局变量❌ 进程有独立内存空间，不共享全局变量
忘记 if __name__ == "__main__"Windows 上会报错，必须添加
multiprocessing.Queue 在进程池中可用❌ 使用 Manager().Queue 替代
join() 前不调用 close()✅ 必须先 close() 再 join()
进程池中异步结果不调用 get()不调用 get() 可能丢失结果或异常

📌 核心记忆：

  • 进程是资源分配的基本单位，有独立内存空间
  • 多进程不受 GIL 限制，适合 CPU 密集型任务
  • Windows 上必须使用 if __name__ == "__main__": 保护
  • 进程间不共享全局变量，使用 Queue 通信
  • 进程池中使用 Manager().Queue 替代 multiprocessing.Queue
  • 进程池使用步骤：创建 → 提交任务 → close() → join()
