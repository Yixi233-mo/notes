一、作用域（Scope） ​

作用域是Python代码中的一段文本区域，从这个区域可以直接访问该命名空间。

1.1 四种作用域（LEGB规则） ​

层级名称说明搜索顺序
LLocal（局部）函数内部定义的名称第1位
EEnclosing（嵌套）外层闭包函数中的名称第2位
GGlobal（全局）当前模块的全局名称第3位
BBuilt-in（内置）Python内置名称第4位
plain
LEGB 查找规则：
L → E → G → B
（从最内层逐层向外搜索）

1.2 作用域示例 ​

python
x = "global"        # G: 全局作用域

def outer():
    x = "enclosing"   # E: 嵌套作用域
    
    def inner():
        x = "local"   # L: 局部作用域
        print(x)      # 查找顺序: local → enclosing → global → built-in
    
    inner()

outer()   # 输出: local

1.3 global 和 nonlocal 关键字 ​

python
# global：在局部作用域中修改全局变量
count = 0

def increment():
    global count
    count += 1

increment()
print(count)   # 1

# nonlocal：在内部作用域中修改嵌套作用域的变量
def outer():
    x = 10
    
    def inner():
        nonlocal x
        x += 5
    
    inner()
    print(x)   # 15

outer()

二、闭包（Closure） ​

2.1 什么是闭包 ​

闭包是指一个函数对象，它保留了对其定义时所在作用域中变量的引用，即使该作用域已经执行完毕。

特点说明
数据持久化函数内的数据在函数调用后仍然存在
数据隐藏避免使用全局变量，隐藏数据
工厂函数可以创建定制化的函数

2.2 构建闭包的条件 ​

  1. 外部函数内定义一个内部函数
  2. 内部函数使用了外部函数的变量
  3. 外部函数将内部函数作为返回值返回

2.3 闭包示例 ​

python
def linear(a, b):
    """外部函数：创建一次线性函数"""
    def inner(x):
        """内部函数：计算 a*x + b"""
        return a * x + b
    return inner

# 创建不同的线性函数
y1 = linear(1, 1)   # y = x + 1
y2 = linear(2, 3)   # y = 2x + 3

print(y1(5))   # 6（1*5 + 1）
print(y2(5))   # 13（2*5 + 3）

三、装饰器（Decorator） ​

3.1 什么是装饰器 ​

装饰器允许在不修改原有函数代码的基础上，动态地增加或修改函数的功能。

本质说明
输入一个函数
输出一个新的包装后的函数
目的在不修改原函数的情况下增强功能

3.2 基本装饰器实现 ​

python
from math import sqrt

# 装饰器函数
def decorator(func):
    def inner(x):
        x = abs(x)      # 添加功能：取绝对值
        return func(x)  # 调用原函数
    return inner

# 手动应用装饰器
def sqrt_func(x):
    return sqrt(x)

sqrt_func = decorator(sqrt_func)
print(sqrt_func(-4))   # 2.0

四、多层装饰器 ​

多个装饰器的执行顺序：从下往上（离函数最近的先执行）。

python
from math import sqrt

# 装饰器1：将参数转换为整数
def get_integer(func):
    def inner(x):
        x = int(x)
        return func(x)
    return inner

# 装饰器2：将参数转换为非负数
def get_absolute(func):
    def inner(x):
        x = abs(x)
        return func(x)
    return inner

@get_integer      # 后执行（外层）
@get_absolute     # 先执行（内层）
def sqrt_func(x):
    return sqrt(x)

print(sqrt_func("-4"))   # 2.0

五、带参数的装饰器 ​

python
from math import sqrt

# 带参数的装饰器
def times(n):
    """返回一个装饰器，对结果应用 n 次函数"""
    def get_absolute(func):
        def inner(x):
            x = abs(x)
            for _ in range(n):
                x = func(x)
            return x
        return inner
    return get_absolute

@times(2)   # 对结果开两次根号
def sqrt_func(x):
    return sqrt(x)

print(sqrt_func(-16))   # 2.0（sqrt(sqrt(16)) = sqrt(4) = 2）

六、类装饰器 ​

类装饰器通过实现 __call__() 方法，使得类的实例可以像函数一样被调用。

python
from math import sqrt

class DecoratorClass:
    def __init__(self, func):
        self.func = func
    
    def __call__(self, x):
        """当实例被调用时执行"""
        x = abs(x)           # 添加功能
        return self.func(x)  # 调用原函数

@DecoratorClass
def sqrt_func(x):
    return sqrt(x)

print(sqrt_func(-4))   # 2.0

七、装饰器综合示例 ​

python
"""
日志装饰器：记录函数调用信息
"""

import time
from functools import wraps

def logger(level="INFO"):
    """带参数的日志装饰器"""
    def decorator(func):
        @wraps(func)   # 保留原函数的元信息
        def wrapper(*args, **kwargs):
            # 记录调用时间
            timestamp = time.strftime("%Y-%m-%d %H:%M:%S")
            
            # 记录参数
            args_str = ", ".join(map(str, args))
            kwargs_str = ", ".join([f"{k}={v}" for k, v in kwargs.items()])
            params = args_str
            if kwargs_str:
                params += f", {kwargs_str}" if params else kwargs_str
            
            # 执行前日志
            print(f"[{timestamp}] [{level}] 调用 {func.__name__}({params})")
            
            # 执行原函数
            start_time = time.time()
            result = func(*args, **kwargs)
            end_time = time.time()
            
            # 执行后日志
            print(f"[{timestamp}] [{level}] {func.__name__} 执行完成，耗时 {end_time - start_time:.4f}s")
            print(f"[{timestamp}] [{level}] 返回值: {result}")
            print("-" * 50)
            
            return result
        return wrapper
    return decorator


# 使用装饰器
@logger(level="DEBUG")
def calculate(a, b, operator="+"):
    """计算器函数"""
    if operator == "+":
        return a + b
    elif operator == "-":
        return a - b
    elif operator == "*":
        return a * b
    elif operator == "/":
        return a / b
    else:
        raise ValueError(f"不支持的运算符: {operator}")

@logger(level="INFO")
def fibonacci(n):
    """斐波那契数列"""
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)


if __name__ == "__main__":
    calculate(10, 5, "*")
    calculate(100, 3, operator="/")
    fibonacci(5)

八、functools.wraps 的作用 ​

使用 @wraps 可以保留原函数的元信息（如 __name__、__doc__）：

python
from functools import wraps

def decorator(func):
    @wraps(func)   # 保留原函数信息
    def inner(*args, **kwargs):
        return func(*args, **kwargs)
    return inner

@decorator
def my_function():
    """这是我的函数"""
    pass

print(my_function.__name__)   # my_function（保留原名）
print(my_function.__doc__)    # 这是我的函数（保留文档）

九、易错点总结 ​

易错点正确理解
装饰器在函数定义时执行✅ 装饰器在 @ 语法糖处立即执行
闭包和装饰器是同一概念❌ 闭包是基础，装饰器是闭包的应用
多层装饰器执行顺序从上到下❌ 从下往上执行（离函数最近的先执行）
装饰器会修改原函数❌ 装饰器不修改原函数，而是返回新的函数
忘记使用 functools.wraps会导致函数元信息丢失

# === 作用域 ===

📌 核心记忆：

  • 作用域：LEGB 规则（Local → Enclosing → Global → Built-in）
  • 闭包：内部函数引用外部变量，外部函数返回内部函数
  • 装饰器：不修改原函数，动态增强功能
  • @** 语法糖**：@decorator 等价于 func = decorator(func)
  • 多层装饰器：从下往上执行
  • @wraps：保留原函数的元信息（名称、文档等）
